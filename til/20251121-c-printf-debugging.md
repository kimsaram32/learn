# 20251121 - C printf debugging 시 주의점 (short)

C에서 빠른 디버깅을 위해 `printf`를 찍어보면서 문제의 원인을 찾는 경우가 많았다.

그러나 에러 등으로 인해 프로그램의 강제 종료가 동반되는 경우,
이것이 제대로 작동하지 않을 수 있다.

기본적으로 `printf` 호출은 바로 write 연산을 수행하지 않고 buffering이 일어난다.
일반적으론 flush 가 곧 일어나기에 화면에 바로 찍히는 것이다.

그러나 printf 직후 프로그램이 강제 종료되는 경우엔 flush가 일어나지 않아
printf의 결과물이 실제로는 출력되지 않게 된다.

이러면 문제의 원인을 찾기 어려워지거나 잘못 파악하게 된다.
