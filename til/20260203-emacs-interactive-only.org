* 20260203 - Elisp interactive-only functions (short)
** Interactive-only
Elisp의 함수들 중 일부는 command로써 사용자가 호출 가능하다. 이렇게
사용되는 함수를 interactive 하다고 한다. 이때 특정 함수가 interactive
하게만 잘 작동하도록 설계된 경우, 이를 나타낼 수 있는 방법은 함수에
interactive-only 속성을 부여하는 것이다.

#+begin_src elisp :results none
  (defun test/interactive-only-function ()
    (interactive)
    (declare (interactive-only t))
    (message "Hi from intractive only function"))
#+end_src

byte compiler는 interactive-only한 함수가 Elisp에서 실행되었을 경우
경고를 띄워준다.
** Declare
=declare= 는 Elisp에서의 매크로이며, 기본적으론 인수를 무시하도록 한다
(즉 아무 동작도 실행되지 않도록 만든다). 그러나 function definition
등에서 사용될 경우, =declare= 에 넘겨진 인수를 통해 함수 등에 속성을
지정해줄 수 있다.

이 동작은 declare 자체적으로 이루어지는 것이 아니라, =defun= 등 관련
매크로에서 special behavior 형태로 구현하는 것이라고 한다.
